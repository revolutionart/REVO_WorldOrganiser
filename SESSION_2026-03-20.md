### 4. Editor Mode Icon Registration — Investigation & Fix

**Background**: The user noticed the icon was not appearing in the Selection Mode dropdown and asked whether the plugin should be migrated to the newer UE5 `UEdMode` pattern (used by REVO Wall Builder) which sets `Info = FEditorModeInfo(...)` in the constructor.

**Attempted migration (rolled back)**:
- Moved `Info = FEditorModeInfo(...)` into `FOrganisationEdMode` constructor with icon set.
- Simplified `RegisterMode` to pass only the mode ID.

**Problem with the attempt**: `FEditorModeInfo` in the constructor is the **`UEdMode`** (UObject-based) pattern. This plugin uses legacy **`FEdMode`**. For `FEdMode`, all metadata (display name, icon, visibility) must be passed directly to `FEditorModeRegistry::Get().RegisterMode<>()`. Dropping them caused `bVisible` to default to `false`, hiding the mode entirely from the dropdown.

**Final fix**: Restored the correct `FEdMode` registration with:
```cpp
FEditorModeRegistry::Get().RegisterMode<FOrganisationEdMode>(
    FOrganisationEdMode::EM_OrganisationModeId,
    FText::FromString("REVO World Organiser"),
    FSlateIcon(FREVOWorldOrganiserStyle::GetStyleSetName(), "REVOWorldOrganiser.ToolbarIcon"),
    true);
```

**Note on UEdMode migration**: Migrating from `FEdMode` to `UEdMode` is a significant refactor (different base class, toolkit type, registration macros, lifecycle signatures) and was deferred as a separate task. The current `FEdMode` implementation works correctly.

**Files changed**: `REVOWorldOrganiser.cpp`, `OrganisationEdMode.cpp` (reverted to `= default` constructor)
