## Issues Addressed & Solutions

### 1. **Parent Class Support in Exclude Filters**
**Problem**: Parent classes (e.g., PackedLevelActor, BP Custom Defered Decal) were missing from exclude filter dropdowns.

**Root Cause**: `RefreshClassPickerOptions()` only collected exact classes, not parent classes in the inheritance hierarchy.

**Solution**: Modified class collection to walk the inheritance chain using `UClass::GetSuperClass()`:
```cpp
// Collect actor class AND all parent classes in inheritance hierarchy
UClass* ActorClass = Actor->GetClass();
while (ActorClass && ActorClass != AActor::StaticClass())
{
    UniqueActorClasses.Add(ActorClass);
    ActorClass = ActorClass->GetSuperClass();
}
```

**File Modified**: `OrganisationEdModeToolkit.cpp` - `RefreshClassPickerOptions()`

---

### 2. **Preview/Selection Count Mismatch**
**Problem**: Preview reported 113 actors would be selected, but actual selection returned 0 actors.

**Root Cause**: `RunSelection()` was missing validation check for locked actors/layers that `RunPreview()` had.

**Solution**: Added `GEditor->CanSelectActor()` check in `RunSelection()`:
```cpp
// Only select actors that can actually be selected (not locked, not in locked layers, etc.)
if (!GEditor->CanSelectActor(Actor, true, false))
{
    ++DebugFailedCanSelect;
    continue;
}
```

**Files Modified**: `SelectActorsInVolume.cpp` - `RunSelection()`

---

### 3. **Containment Algorithm Too Lenient**
**Problem**: "Contains Fully" mode was selecting actors that were only 10cm inside the volume (using 75% overlap threshold).

**Root Cause**: Original algorithm used percentage-based overlap check, not true containment.

**Solution**: Implemented strict 8-corner boundary checking:
```cpp
case EVolumeSelectionMode::ContainsFully:
{
    // True containment check: all 8 corners must be inside volume
    return VolumeBounds.IsInside(ActorBounds.Min) && 
           VolumeBounds.IsInside(ActorBounds.Max) &&
           VolumeBounds.IsInside(FVector(ActorBounds.Min.X, ActorBounds.Min.Y, ActorBounds.Max.Z)) &&
           VolumeBounds.IsInside(FVector(ActorBounds.Min.X, ActorBounds.Max.Y, ActorBounds.Min.Z)) &&
           VolumeBounds.IsInside(FVector(ActorBounds.Max.X, ActorBounds.Min.Y, ActorBounds.Min.Z)) &&
           VolumeBounds.IsInside(FVector(ActorBounds.Min.X, ActorBounds.Max.Y, ActorBounds.Max.Z)) &&
           VolumeBounds.IsInside(FVector(ActorBounds.Max.X, ActorBounds.Min.Y, ActorBounds.Max.Z)) &&
           VolumeBounds.IsInside(FVector(ActorBounds.Max.X, ActorBounds.Max.Y, ActorBounds.Min.Z));
}
```

**Files Modified**: `SelectActorsInVolume.cpp` - `PassesVolumeFilter()`

### 4. **Tag Filtering Dependency Bug**
**Problem**: Tag filters (e.g., "StripeDecal" tag added to 24 actors) returned 0 selections unless Actor Type filters were also active.

**Root Cause**: Filter logic incorrectly required type/class filters to be active before evaluating tags.

**Solution**: Restructured filter validation to allow tags as standalone filter:
```cpp
// Validate filter configuration
const bool bHasActorTypeFilter = Settings.ActorTypeMask != 0;
const bool bHasClassPickerFilter = Settings.bEnableClassPickerFilter && 
    (!Settings.SelectedActorClassPath.IsEmpty() || !Settings.SelectedComponentClassPath.IsEmpty());
const bool bHasTagFilter = Settings.IncludeActorTags.Num() > 0;

if (!bHasActorTypeFilter && !bHasClassPickerFilter && !bHasTagFilter)
{
    // Show error: no filters active
    return 0;
}

// Apply type/class filters (OR logic: pass if either matches)
bool bPassesTypeOrClassFilter = false;
if (bHasActorTypeFilter && PassesActorTypeFilter(Actor, Settings.ActorTypeMask))
{
    bPassesTypeOrClassFilter = true;
}
if (bHasClassPickerFilter && PassesClassPickerFilter(Actor, Settings))
{
    bPassesTypeOrClassFilter = true;
}

// If type/class filters are active but actor doesn't match, reject (unless only using tags)
if ((bHasActorTypeFilter || bHasClassPickerFilter) && !bPassesTypeOrClassFilter)
{
    continue;
}
```

**Files Modified**: `SelectActorsInVolume.cpp` - `RunPreview()`, `RunSelection()`

### 5. **Epsilon Expansion Bug**
**Problem**: Epsilon tolerance was expanding the volume bounds, allowing partially-outside actors to pass "Contains Fully" checks.

**Root Cause**: Volume bounds were being expanded by epsilon before containment check.

**Solution**: Removed epsilon expansion from ContainsFully mode - use exact boundary checking without tolerance.

**Files Modified**: `SelectActorsInVolume.cpp` - `PassesVolumeFilter()`

### 7. **Child Actor Counting (MAJOR FIX)**
**Problem**: Preview showed 247 actors, but World Outliner showed 269 selected actors.

**Root Cause**: Unreal Engine automatically selects child/attached actors when parent is selected. Our code was only counting explicit selections (247), not Unreal's automatic additions (22 child actors).

**Solution**: 

**A) RunSelection() Enhancement**:
```cpp
GEditor->NoteSelectionChange();

// Count actual selection including auto-selected children/attached actors
int32 ActualSelectionCount = 0;
for (FSelectionIterator It(*GEditor->GetSelectedActors()); It; ++It)
{
    if (Cast<AActor>(*It))
    {
        ActualSelectionCount++;
    }
}

if (ActualSelectionCount != SelectedCount)
{
    UE_LOG(LogTemp, Warning, TEXT("ACTUAL SELECTION (with children): %d"), ActualSelectionCount);
}

FNotificationInfo Info(FText::Format(
    LOCTEXT("SelectionComplete", "Selected {0} actor(s) ({1} with attached/child actors)"),
    FText::AsNumber(SelectedCount),
    FText::AsNumber(ActualSelectionCount)
));
```

**B) RunPreview() Enhancement** (Temporary Selection Simulation):
```cpp
// Save current selection
TArray<AActor*> OriginalSelection;
for (FSelectionIterator It(*GEditor->GetSelectedActors()); It; ++It)
{
    if (AActor* SelectedActor = Cast<AActor>(*It))
    {
        OriginalSelection.Add(SelectedActor);
    }
}

// Clear selection and select preview actors to count with children
GEditor->SelectNone(false, false);

// Re-apply all filters and select matching actors
for (TActorIterator<AActor> It(World); It; ++It)
{
    // ... filter logic ...
    GEditor->SelectActor(Actor, true, false);
}

GEditor->NoteSelectionChange(); // Triggers automatic child selection

// Count total selection including auto-selected children
int32 PreviewCountWithChildren = 0;
for (FSelectionIterator It(*GEditor->GetSelectedActors()); It; ++It)
{
    if (Cast<AActor>(*It))
    {
        PreviewCountWithChildren++;
    }
}

// Restore original selection
GEditor->SelectNone(false, false);
for (AActor* Actor : OriginalSelection)
{
    if (IsValid(Actor))
    {
        GEditor->SelectActor(Actor, true, false);
    }
}
GEditor->NoteSelectionChange();

// Show notification with both counts
FNotificationInfo Info(FText::Format(
    LOCTEXT("PreviewComplete", "Preview: {0} actor(s) match filters ({1} with attached/child actors)"),
    FText::AsNumber(PreviewCount),
    FText::AsNumber(PreviewCountWithChildren)
));
```

**Files Modified**: `SelectActorsInVolume.cpp` - `RunPreview()`, `RunSelection()`

**Result**: 
- Preview: "Preview: 247 actor(s) match filters (269 with attached/child actors)"
- Selection: "Selected 247 actor(s) (269 with attached/child actors)"
- Both counts now match World Outliner's 269 actors

## Known Behavior

### Child Actor Selection
- Unreal Engine **automatically selects child/attached actors** when parent is selected
- Blueprint actors with nested ChildActorComponents contribute additional selections
- Example: BP_Vending_Machine_Large has 2 child actors → selecting 1 parent = 3 total selections
- This is **expected Unreal Engine behavior**, not a bug
- Our code now accurately reports both counts:
  - **Explicit count**: Actors directly matching filters (247)
  - **Total count**: Including auto-selected children (269)

### Filter Logic (OR + AND)
- **OR logic** between Actor Type filters and Class Picker filters
- **AND logic** for Tag filters (must pass both Include AND Exclude checks)
- **AND logic** for Volume containment (must pass spatial test)
- Tags can work as **standalone filter** without type/class filters
